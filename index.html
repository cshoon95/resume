사용자 경험을 생각하는 개발자

사용자에게 좋은 경험을 전달하는 일에 큰 보람을 느끼기 때문에 사용자 경험을 향상시키는 기회를 갖고 싶어 합니다. 사용자 경험은 기획자, 디자이너의 영역만이 아니라 개발자도 포함된 영역이라고 생각합니다. 개발자인 저는웹 표준을 준수하고 접근성을 고려하여 사용자에게 더 나은 경험과 서비스를 제공하기 위해 노력합니다.


배우는 것을 좋아하는 개발자

새로운 기술 학습에 관심이 많으며, 배우는 것을 두려워하지 않고 좋아합니다. 더 좋은 최신 기술이 있다면 써보고 싶은 욕심이 있어 하루빨리 배워보고 싶어 합니다. 주로 출퇴근 시간에도 개발 관련 강의를 듣거나, 개발 관련 서칭을 합니다. 그리고 퇴근 후 공부한 것을 개인 프로젝트에 적용해 보고는 합니다. 최근에는 SSR기법을 사용하는 Next.js를 간단한 토이프로젝트로 배우고 있습니다.


기록하고 공유하는 개발자

개발을 하다가 다른 사람들에게 공유하면 좋을 것 같은 내용들은 정리해서 블로그에 작성합니다. 블로그에 글을 공개로 게시하면 저 혼자 보는 것이 아니다 보니, 신경쓰고 분석하면서 글을 쓰게 되어서 저한테도 많은 도움이 됩니다. 또한, 시간이 지나고 내용을 다시 확인했을 때 제가 작성한 글이다 보니 기억도 더 잘되고 이해가 더 잘 되는 효과도 있습니다.

공통 컨트롤 마이그레이션

기존에 Javascript와 jQuery 기반으로 작업된 컨트롤과 jQuery 기반의 Validate.js인 Parsley를 Vanila Js로 마이그레이션하는 프로젝트를 진행하였습니다. 기존에 사용된 jQuery의 기능 대부분을 이제는 Javascript만으로도 구현이 가능해짐으로써 jQuery를 제거하여 코드 베이스의 크기와 복잡성을 줄일 수 있었습니다. 사용자가 주로 입력하는 화면의 폼 컨트롤과 유효성 검사를 마이그레이션 하면서 Vanila Js로 커스터마이징하여 프로젝트의 요구 사항에 맞게 수정하다보니 효율적인 유효성 검사를 구현할 수 있었습니다. 이덕분에 유효성 검사 규칙, 오류 메시지, 스타일 및 동작을 조절하여 사용자 경험을 향상시킬 수 있었습니다.

  • Front-end: 2명
  • 공통 컨트롤 및 validation 라이브러리 jQuery -> Vanila JS로 마이그레이션
  • jQuery 유효성 검사 라이브러리(Parsley) Vanila JS로 커스터마이징
  • Javascript, GitLab
